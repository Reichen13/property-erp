## 系统概述

物业ERP系统是一套完整的物业费用管理解决方案，采用复式记账法确保财务数据准确性。

### 核心概念

| 概念 | 说明 |
|------|------|
| 账期 | 格式YYYY-MM，如2026-01 |
| 会计归属期 | 收入确认的会计期间，可与账期不同 |
| 钱包余额 | 业主预存款，可抵扣账单 |
| 复式记账 | 借贷平衡，每笔交易同时记录借方和贷方 |

### 账单状态流转

```
待缴 → 部分已缴 → 已缴
  ↓
作废
```

## 管理员使用手册

### 角色定位
系统最高权限用户，负责系统配置、用户管理、数据维护。

### 可访问功能
- 所有功能模块

### 核心职责

#### 1. 用户管理
**路径**：系统与运维 → 系统管理 → 用户管理

操作步骤：
1. 查看现有用户列表
2. 点击"新增用户"
3. 填写账号、密码
4. 选择角色（管理员/集团财务/项目财务/审批员）
5. 选择归属物业（集团财务可选"无/全局"）
6. 点击"添加用户"

**注意事项**：
- 密码至少6位
- 项目财务必须绑定具体物业项目
- 集团财务可不绑定物业，查看所有项目数据

#### 2. 费用科目管理
**路径**：系统与运维 → 系统管理 → 费用科目

常见科目设置：
| 科目名称 | 税率 |
|----------|------|
| 物业费 | 6% |
| 停车费 | 6% |
| 水费 | 3% |
| 电费 | 3% |

#### 3. 系统初始化
**路径**：系统与运维 → 系统初始化

首次使用系统时：
1. 创建数据库索引（提升查询性能）
2. 检查账户科目是否完整

#### 4. 数据备份
**路径**：报表与备份 → 数据备份

- 系统每日首次登录自动备份
- 重要操作前建议手动备份
- 备份文件存储在 `backups/` 目录

#### 5. 清除测试数据
**路径**：系统与运维 → 清除测试数据

**⚠️ 危险操作**：
- 仅在测试环境使用
- 会删除所有业务数据
- 保留系统配置和管理员账号
- 需输入确认文字才能执行

## 集团财务使用手册

### 角色定位
集团层面的财务监督人员，主要负责数据审核、报表分析、审计追踪。以**只读查询**为主，不直接操作业务数据。

### 可访问功能
| 功能模块 | 操作权限 |
|----------|----------|
| 运营驾驶舱 | 查看 |
| 数据中心 | 查看、导出 |
| 财务报表 | 查看、导出 |
| 收费核对 | 查看 |
| 三方核对 | 查看 |
| 财务检查 | 查看 |
| 审计查询 | 查看 |
| 变更历史 | 查看 |
| 收款对账 | 查看 |
| 欠费追踪 | 查看 |

### 日常工作流程

#### 一、每日工作

##### 1. 查看运营驾驶舱
**路径**：核心业务 → 运营驾驶舱

关注指标：
- 期间实收：当期实际收款金额
- 期间折扣：当期减免金额
- 期间新增欠费：当期新产生的欠费
- 预存余额：所有业主预存款总和

**操作步骤**：
1. 设置查询日期范围
2. 查看各项指标
3. 对比历史数据，发现异常

**异常处理**：
- 实收金额异常波动 → 查看收款对账明细
- 减免金额过高 → 查看审计日志，确认审批流程

##### 2. 查看收款对账
**路径**：报表与备份 → 收款对账

**操作步骤**：
1. 选择日期范围
2. 查看按支付方式统计
3. 核对收款总额

**核对要点**：
- 微信/支付宝金额与第三方平台对账
- 现金金额与出纳日报核对
- 银行转账与银行流水核对

#### 二、周度工作

##### 1. 欠费追踪分析
**路径**：报表与备份 → 欠费追踪

**关注内容**：
- 欠费总额趋势
- 欠费房产数量
- 欠费排行榜（Top 20）

**分析要点**：
1. 欠费集中在哪些房产？
2. 欠费时间多长？
3. 是否需要催收？

##### 2. 审计日志检查
**路径**：数据与审计 → 审计查询

**检查重点**：
- 大额减免操作
- 账单作废操作
- 数据修改操作

**操作步骤**：
1. 选择时间范围（最近7天）
2. 筛选操作类型
3. 查看操作详情
4. 使用trace_id追踪关联操作

#### 三、月度工作

##### 1. 财务报表分析
**路径**：报表与备份 → 财务报表

**利润表查看**：
1. 设置会计期间范围
2. 查看应收收入、减免金额、已收款金额
3. 计算收缴率

**账期对比分析**：
1. 选择两个会计期进行对比
2. 分析应收、实收、收缴率变化
3. 找出差异原因

##### 2. 三方核对
**路径**：数据与审计 → 三方核对

**核对内容**：
1. 房产余额总和 vs 预收账款科目余额
2. 账单应收总额
3. 收款记录按支付方式统计

**差异处理**：
- 差异<0.01元：正常，忽略
- 差异>0.01元：需排查原因，联系项目财务

##### 3. 财务勾稽检查
**路径**：数据与审计 → 财务检查

**检查项目**：
| 检查项 | 正常状态 | 异常处理 |
|--------|----------|----------|
| 负余额房产 | 无 | 联系项目财务核实 |
| 超额缴费 | 无 | 检查是否误操作 |

##### 4. 数据导出
**路径**：核心业务 → 数据中心 → 数据导出

可导出内容：
- 账单明细CSV
- 资金流水CSV

**用途**：
- 制作月度财务报告
- 与其他系统数据核对
- 存档备查

#### 四、变更历史查询
**路径**：数据与审计 → 变更历史

**使用场景**：
- 追溯数据修改记录
- 审计需要时查询
- 排查数据问题

**查询方法**：
1. 选择数据表（rooms/bills/payment_records）
2. 输入记录ID（可选）
3. 选择时间范围
4. 查看变更详情（原值、新值、操作人、原因）

### 注意事项

1. **只读原则**：集团财务主要是监督审核，不直接修改业务数据
2. **定期检查**：建议每日查看驾驶舱，每周检查审计日志
3. **异常上报**：发现数据异常及时与项目财务沟通
4. **数据保密**：导出的数据注意保密，不外传

### 常见问题

**Q1：看到的数据是所有项目的吗？**
A：是的，集团财务可以查看所有物业项目的汇总数据。

**Q2：发现数据异常怎么办？**
A：
1. 先通过审计日志查看操作记录
2. 联系相关项目财务核实
3. 如需修改，由项目财务或管理员操作

**Q3：如何导出特定时间段的数据？**
A：在数据中心设置筛选条件后导出，或在财务报表中设置会计期间。

### 系统初始化（首次使用）

#### 第一步：导入房产档案
**路径**：核心业务 → 资源档案 → 批量导入

**准备工作**：
1. 下载导入模板
2. 按模板格式整理数据

**模板字段说明**：
| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 房号 | ✓ | 唯一标识 | 1-101 |
| 业主 | ✓ | 业主姓名 | 张三 |
| 业主电话 | | 联系方式 | 13800138000 |
| 面积 | | 建筑面积 | 89.5 |
| 费用项目 | | 费用科目名称 | 物业费 |
| 项目月标准金额 | | 每月应收金额 | 200 |
| 历史欠费 | | 期初欠费金额 | 1200 |
| 欠费周期起 | | 欠费开始日期 | 2025-07-01 |
| 欠费周期终 | | 欠费结束日期 | 2025-12-31 |
| 预缴金额 | | 期初预存余额 | 500 |
| 已缴金额 | | 已收款金额（可选） | 0 |
| 减免金额 | | 已减免金额（可选） | 0 |
| 会计归属期 | | 收入归属期（可选） | 2025-07 |

**导入步骤**：
1. 勾选"先试运行(Dry-run)"
2. 上传文件，预览数据
3. 确认无误后，取消勾选"先试运行"
4. 点击"开始导入"
5. 记录批次ID（用于回滚）

**⚠️ 注意事项**：
- 同一房号多个费用项目需分多行
- 历史欠费会自动生成账单
- 预缴金额会增加到钱包余额
- 导入前建议备份数据库

**问题处理**：
- 导入失败：检查数据格式，特别是日期格式
- 数据错误：使用批次ID回滚，修正后重新导入

#### 第二步：核对导入数据
1. 在"查询/维护"标签页搜索房号
2. 确认业主信息、费用标准正确
3. 检查账单是否正确生成

### 日常工作流程

#### 一、每日开始工作

##### 1. 查看运营驾驶舱
**路径**：核心业务 → 运营驾驶舱

快速了解：
- 今日/本月收款情况
- 当前欠费总额
- 预存余额总额

##### 2. 检查待处理事项
- 待缴费账单
- 待审批减免申请（如有）

#### 二、收银操作

##### 场景1：业主充值
**路径**：核心业务 → 收银台

**操作步骤**：
1. 搜索/选择房号
2. 展开"钱包充值"
3. 输入充值金额
4. 选择收款方式
5. 点击"确认充值"

**系统自动执行**：
- 增加房产余额
- 生成收款记录
- 记录复式分录
- 写入审计日志

**注意事项**：
- 确认金额无误再提交
- 收款方式要与实际一致
- 充值后打印收据给业主

##### 场景2：账单缴费
**路径**：核心业务 → 收银台

**操作步骤**：
1. 选择房号
2. 查看"待缴费账单"列表
3. 勾选要缴费的账单
4. 选择支付方式：
   - **余额抵扣**：从钱包扣款
   - **微信/支付宝**：现场收款
5. 点击"确认支付"

**支付方式选择**：
| 方式 | 适用场景 | 注意事项 |
|------|----------|----------|
| 余额抵扣 | 业主有预存款 | 余额不足会提示 |
| 微信/支付宝 | 现场扫码支付 | 确认到账后再操作 |

**部分缴费**：
- 可只勾选部分账单
- 账单状态变为"部分已缴"

##### 场景3：批量缴费
**路径**：数据与审计 → 批量操作 → 批量缴费

**适用场景**：
- 多个房产同时缴费
- 银行代扣批量处理

**操作步骤**：
1. 选择账期
2. 勾选要缴费的房产
3. 选择支付方式
4. 点击"批量缴费"

#### 三、账单管理

##### 1. 批量生成账单
**路径**：核心业务 → 财务管理 → 批量生成账单

**操作步骤**：
1. 选择生成依据：
   - **按档案预设金额**：使用房产的收费标准
   - **按单价x面积**：输入单价，系统计算
2. 选择费用类型或勾选"生成所有费用项目"
3. 输入账期（如2026-01）
4. 点击"全量生成"

**生成时机**：
- 每月初生成当月账单
- 建议在1-5号完成

**注意事项**：
- 已存在的账单不会重复生成
- 生成后检查账单数量和金额

##### 2. 月度关账
**路径**：核心业务 → 财务管理 → 月度关账

**关账前检查**：
1. 所有账单已生成
2. 收费核对无异常
3. 三方核对平衡

**操作步骤**：
1. 输入账期
2. 点击"关账"

**关账后**：
- 该账期数据锁定
- 不能新增/修改账单
- 不能进行收款操作

**解锁账期**：
- 如需修改已关账数据
- 点击"解锁"按钮
- 修改完成后重新关账

##### 3. 发票管理
**路径**：核心业务 → 财务管理 → 发票管理

**开票条件**：
- 账单状态为"已缴"
- 尚未开过票

**操作步骤**：
1. 选择要开票的账单
2. 输入发票编号
3. 输入发票抬头
4. 点击"开具发票"

#### 四、收费核对

##### 1. 日常核对
**路径**：数据与审计 → 收费核对

**核对内容**：
- 按账期汇总应收/实收
- 按费用类型分类统计
- 查看收缴率

**操作步骤**：
1. 选择账期
2. 选择费用类型（或全部）
3. 查看汇总数据
4. 检查明细

##### 2. 三方核对
**路径**：数据与审计 → 三方核对

**核对项目**：
1. 房产余额 vs 预收账款科目
2. 账单应收总额
3. 收款记录统计

**差异处理**：
- 差异<0.01元：正常舍入误差
- 差异>0.01元：需排查原因

##### 3. 财务检查
**路径**：数据与审计 → 财务检查

**检查项目**：
- 负余额房产
- 超额缴费账单

#### 五、资源档案维护

##### 1. 新增房产
**路径**：核心业务 → 资源档案 → 入伙/新增

**操作步骤**：
1. 输入房号
2. 输入业主信息
3. 添加费用项目和标准金额
4. 点击"添加"

##### 2. 修改房产信息
**路径**：核心业务 → 资源档案 → 查询/维护

**可修改内容**：
- 业主姓名
- 联系电话
- 费用标准

##### 3. 批次回滚
**路径**：核心业务 → 资源档案 → 批次回滚

**使用场景**：
- 批量导入数据有误
- 需要撤销整批导入

**操作步骤**：
1. 输入批次ID
2. 点击"回滚执行"

**⚠️ 注意**：只回滚账单，不回滚房产档案

### 月度结账流程

```
第1步：生成当月账单
    ↓
第2步：日常收款
    ↓
第3步：收费核对
    ↓
第4步：三方核对
    ↓
第5步：财务检查
    ↓
第6步：关账
```

### 注意事项

1. **操作顺序**：
   - 先导入档案，再生成账单
   - 先收款，再关账
   - 关账前完成所有核对

2. **数据准确性**：
   - 收款金额确认后再操作
   - 支付方式要与实际一致
   - 定期核对数据

3. **异常处理**：
   - 操作失败及时记录错误信息
   - 数据问题先排查再处理
   - 重大问题联系管理员

4. **安全规范**：
   - 不要共享账号
   - 定期修改密码
   - 离开时锁定电脑

### 常见问题

**Q1：账单金额错误怎么办？**
A：
1. 账期未关账：直接修改账单
2. 账期已关账：先解锁账期，修改后重新关账

**Q2：业主要求退款怎么处理？**
A：
1. 进入财务管理 → 减免审批
2. 创建减免申请
3. 等待审批员审批
4. 审批通过后系统自动调整

**Q3：充值后发现金额错误？**
A：
1. 联系管理员处理
2. 通过调账或减免方式修正
3. 记录在审计日志中

**Q4：导入数据后发现有误？**
A：
1. 使用批次ID回滚账单
2. 修正Excel数据
3. 重新导入

**Q5：三方核对有差异怎么办？**
A：
1. 检查是否有未完成的操作
2. 查看审计日志找异常操作

## 审批员使用手册

### 角色定位
负责减免申请和调账申请的审批工作，确保费用减免的合规性。

### 可访问功能
| 功能模块 | 操作权限 |
|----------|----------|
| 运营驾驶舱 | 查看 |
| 财务管理 | 仅减免审批 |

### 日常工作流程

#### 1. 查看运营驾驶舱
**路径**：核心业务 → 运营驾驶舱

了解整体收费情况，为审批提供参考。

#### 2. 处理减免审批
**路径**：核心业务 → 财务管理 → 减免审批

**审批流程**：
1. 查看待审核申请列表
2. 点击展开查看详情：
   - 申请金额
   - 申请人
   - 申请理由
   - 关联账单信息
3. 审核判断：
   - **通过**：点击"通过"按钮
   - **拒绝**：点击"拒绝"按钮

**审批原则**：
| 情况 | 建议处理 |
|------|----------|
| 金额小、理由充分 | 通过 |
| 金额大、需核实 | 暂缓，联系申请人 |
| 理由不充分 | 拒绝 |
| 重复申请 | 拒绝 |

**审批后**：
- 通过：系统自动调整账单减免金额
- 拒绝：申请状态变为"已拒绝"

### 注意事项

1. **及时处理**：减免申请应在1-2个工作日内处理
2. **审慎审批**：大额减免需谨慎，必要时与管理层沟通
3. **记录留痕**：所有审批操作都有审计日志
4. **合规性**：确保减免符合公司政策

### 常见问题

**Q1：如何查看历史审批记录？**
A：审批员暂无查看历史记录权限，如需查看请联系集团财务或管理员。

**Q2：误操作审批通过怎么办？**
A：联系管理员处理，通过调账方式修正。

**Q3：申请信息不完整怎么办？**
A：拒绝申请，并告知申请人补充信息后重新提交。

## 常见问题与故障排除

### 登录问题

**Q：忘记密码怎么办？**
A：联系管理员重置密码。

**Q：账号被锁定怎么办？**
A：连续输错密码会锁定，等待几分钟后重试，或联系管理员解锁。

### 数据问题

**Q：数据显示不正确？**
A：
1. 刷新页面
2. 检查筛选条件
3. 确认数据是否已提交

**Q：操作后数据没变化？**
A：
1. 检查是否有错误提示
2. 刷新页面重新查看
3. 查看审计日志确认操作是否成功

### 系统问题

**Q：页面加载很慢？**
A：
1. 检查网络连接
2. 减少查询数据量
3. 联系管理员检查服务器

**Q：系统报错怎么办？**
A：
1. 记录错误信息
2. 刷新页面重试
3. 联系管理员处理

### 业务问题

**Q：如何处理历史欠费？**
A：通过资源档案批量导入功能，在"历史欠费"字段填入金额。

**Q：如何处理跨年度账单？**
A：使用"会计归属期"字段指定收入确认的会计期间。

**Q：如何查看某业主的所有记录？**
A：在数据中心或收银台搜索房号查看。
